# NBHAS Development Principles

1. KISS.
2. The Knowledge Base is the single source of truth.
3. Never hard-code business rules.
4. Never edit Assessment Sessions after completion.
5. Never reuse IDs.
6. GitHub manages version history.
7. Build customer value first.
8. Every screen should answer one question.
9. Prefer configuration over code.
10. If adding a new object doesn't reduce complexity, don't add it.
